# chan.py 技术文档

## 一、项目概述

`chan.py` 是基于**缠中说禅技术分析理论**（简称"缠论"）的 Python 参考实现，约 3,600 行代码。它从原始 K 线数据出发，经过多层分析流水线，逐级产出合并 K 线、分型、笔、线段、中枢等金融市场结构，支撑多级别联立分析。

**跨语言验证**：本 Python 实现与 C99 移植版（`chan.c`）在 50,003 根 BTC/USD 5 分钟 K 线上产生完全一致的输出——笔 2,796 根、线段 374 条、中枢 48 个，逐行比对全部匹配。

---

## 二、核心设计思想

### 2.1 变动即重建（Rebuild on Change）

整个系统的基石策略。当一根新 K 线进入系统并引起某层结构失效时，不是"局部修复"，而是将失效部分以后的所有结构**全部弹出**，然后从断裂点**重建**。这保证了一致性：高层结构绝不持有已失效的低层引用。

```
新K线 → 缠论K线重算 → 分型可能失效 → 包含该分型的笔被弹出
                                            ↓
                                    线段可能失效 → 包含该线段的笔被弹出
                                            ↓
                                    中枢可能失效 ...
```

这种策略牺牲了局部运算量，换来了**逻辑正确性**——在任何时刻，系统中的所有结构都是"此刻看到的所有数据"的一致视图。

### 2.2 增量计算（Incremental Computation）

虽然结构是"变动即重建"，但**技术指标**（MACD / RSI / KDJ）采用增量计算——每根新 K 线基于前一根的 EMA/RMA 递推，避免重复扫描全量历史。这保证了实时场景下的计算效率。

### 2.3 双模式管道

观察者内部有两套并行的管道模式：

- **模式一「文-武」**：标准缠论模式。使用特征序列分析，支持缺口 / 小转大 / 老阴老阳等复杂情形。
- **模式二「高-低」**：简化模式（用于扩展线段，类同于**同级别分解**）。只看三线重叠（中枢条件），适合更高层次的同级别分解。

### 2.4 弱引用模型

C 移植版使用区分**强引用**（拥有所有权，影响引用计数）和**弱引用**（裸指针，不影响引用计数）的内存模型。Python 版通过 GC 自然解决，但设计理念一致：高层持有低层引用，低层不反向持有高层引用。在"变动即重建"的清退顺序下，弱引用始终有效（高层先被清空，低层后被释放）。

## 三、完整分析流水线

```
原始K线 ──(K线包含关系合并)──> 缠论K线 ──(三元素分型识别)──> 分型
                                                              │
                                      ┌───────────────────────┘
                                      ↓
                        笔（分型连线，至少5根缠论K线）
                                      │
                    ┌─────────────────┼──────────────────┐
                    ↓                 ↓                   ↓
              笔中枢              线段分析              扩展线段
           (3笔重叠)        (特征序列终结法)        (简化高低模式)
                    │                 │                   │
                    ↓                 ↓                   ↓
              笔中枢序列        线段.中枢序列        扩展线段.中枢序列
                                      │
                                      ↓
                              线段-线段分析
                          (线段视作基础元素，再分析一次)
                                      │
                                      ↓
                              线段.线段中枢序列
```

### 3.1 各层详细说明

#### 第一层：缠论 K 线（合并 K 线）

**作用**：消除 K 线之间的包含关系，为分型识别提供合格的"独立 K 线序列"。

**核心逻辑** (`缠论K线.兼并`, 第 1482 行)：

- 每根新原始 K 线与当前最后一根缠论 K 线比较
- 若不包含 → 创建新缠论 K 线（`添加`模式）
- 若包含且与前缠论 K 线方向不同 → 根据前缠论 K 线方向决定取 `max` 或 `min`，原地合并（`兼并`模式）
- 若包含且与前缠论 K 线方向相同 → 更新位值，保持方向

**关键配置**：
- `缠K合并替换`：`True` 时合并产出新对象（不修改原对象），`False` 时原地修改

#### 第二层：分型

**作用**：从三根相邻缠论 K 线中识别顶分型 / 底分型 / 上升 / 下降 / 扩散五种形态。

**核心逻辑** (`分型结构.分析`, 第 626 行)：

```
左中关系 + 中右关系 → 分型结构
- 向上+向上 → 上升
- 向上+向下 → 顶分型
- 向下+向下 → 下降
- 向下+向上 → 底分型
- 左包含中 或 中包含右 → 包含异常（不允许）
```

分型确定后，会自动计算**分型特征值**（顶分型取中最高，底分型取中最低），并预判下一根缠论 K 线的分型方向。

#### 第三层：笔

**作用**：连接相邻的顶底分型，形成一笔。笔是缠论中最基本的"趋势"单位。

**核心逻辑** (`笔.分析`, 第 1944 行)：

1. **成笔条件**：两分型之间的缠论 K 线数量 ≥ `笔内元素数量`（默认 5）
2. **分型连续**：必须是"顶 → 底"或"底 → 顶"交替
3. **同向覆盖**：新顶高于旧顶 → 弹出旧顶，用新顶；新底低于旧底 → 弹出旧底，用新底
4. **实际极值修正**：笔的端点必须是该区间内的实际最高 / 最低点（而非分型本身的最高 / 最低）
5. **次极值回退**（`笔次级成笔`）：若实际极值不满足包含关系，尝试用次高 / 次低
6. **笔弱化**（`笔弱化`）：允许 3 根 K 线成笔的特殊模式

**关键配置**：
- `笔内元素数量`（默认 5）：最少 K 线数
- `笔内相同终点取舍`（默认 `False` = first）：多个同值终点时选最早
- `笔次级成笔`：启用次高/次低成笔
- `笔弱化` + `笔弱化_原始数量`（默认 3）：弱化成笔的原始 K 线阈值

#### 第四层：线段

**作用**：将连续的笔序列划分为更高级别的"线段"。线段是缠论的核心层级，也是中枢和买卖点的直接载体。

线段分析是整个系统最复杂的部分，因为它需要处理**特征序列**——一种将笔组合为"分型元素"的间接表示，用于判断线段是否终结。

**核心概念：特征序列**

特征序列（`线段特征`）将同方向的连续笔"合并"为一个虚拟的"分型元素"：

- 向上线段中，向下的笔构成特征序列
- 向下线段中，向上的笔构成特征序列
- 连续同向的笔可能被合并（包含关系处理）

**核心逻辑** (`线段.分析`, 第 2726 行)：

1. **初始化**：从笔序列中找到连续三笔满足重叠关系且具有效方向的作为第一个线段
2. **序列重置**：每轮先将线段的基础序列裁剪为笔序列中仍有效的部分
3. **刷新特征序列**：重新计算当前线段内的特征序列分布
4. **判断终结**：当特征序列形成顶/底分型时，线段终结
5. **创建新线段**：将终结后的笔作为新线段的基础序列

**线段终结的四种修正机制**（按优先级）：

| 修正 | 条件 | 行为 |
|------|------|------|
| 缺口突破 | 老阳/老阴状态下，最后一笔低/高突破线段极值 | 回退笔到前一有效线段 |
| 非缺口下穿刺 | 小阳/少阴状态下存在贯穿伤 | 切割基础序列并回退 |
| 缺口后紧急修正 | 前一缺口刚闭合时的异常情况 | 直接创建新线段 |
| 短路修正 | 之后序列 ≥ 6 笔且最后3笔形成有效方向 | 切割为两个新线段 |

**`线段.分割序列` 详解**（第 2355 行）：

这是理解线段"当下状态"的关键方法。它将一个线段的基础序列分割为三个部分：

```python
前, 后, 第三买卖线, 贯穿伤 = 线段.分割序列(段)
```

- **`前`**：从线段起点到终点的笔（不含"之后"的部分）。这是线段已确认的实体部分。
- **`后`**：在线段终点之后的笔。当特征序列形成分型后，线段"武"（终点）之前发生的笔被归入"前"，而"武"之后的笔被归入"后"。
- **第三买卖线**：线段终点与所属中枢的相对位置，用于识别第三类买卖点。
- **贯穿伤**："后"的第一笔即反向突破线段起点的笔。

**`后` 与"未完成的新段"**：当 `后` 序列存在且数量 ≥ 3 时，说明线段终点之后已有足够的笔构成一个新的潜在线段。但由于"变动即重建"策略，这些笔尚未正式形成独立线段——它们处于"待确认"状态，等待特征序列完整终结。此即**未完成的新段**——它已经是分析器内部"下一个线段的胚"，只是还没有被正式创建。

**四象体系**：

| 四象 | 条件 | 含义 |
|------|------|------|
| 小阳 | 向上线段 + 无前一缺口 | 普通的向上 |
| 少阴 | 向下线段 + 无前一缺口 | 普通的向下 |
| 老阳 | 向上线段 + 前一缺口 | 强势向上（但有隐患） |
| 老阴 | 向下线段 + 前一缺口 | 强势向下（但有隐患） |

四象决定特征序列包含关系的处理方式——老阳/老阴时严格处理（包括逆序包含），小阳/少阴时宽松处理。

#### 第五层：中枢

**作用**：三段连续笔 / 线段的**重叠区域**。中枢是缠论判断走势类型的核心结构——任何级别的走势都可以分解为中枢与中枢的连接段。

**核心逻辑** (`中枢.分析`, 第 3185 行)：

1. 扫描虚线序列，找到第一组三线重叠的区间
2. 中枢区间 = 三线的最低高点 ~ 最高低点
3. 后续虚线若与中枢区间重叠 → 加入中枢（中枢延续/扩张）
4. 若虚线脱离中枢区间（缺口） → 检查是否成为第三买卖点 → 可能产生新中枢

**中枢完整性** (`中枢.完整性`, 第 3016 行)：根据《教你炒股票 43》，不完整的中枢大概率会与下一中枢发生**中枢扩展**。

**中枢当前状态** (`中枢.当前状态`, 第 3105 行)：根据《教你炒股票 49》的策略，中枢状态决定操作策略：
- 中枢之中 → 不操作（最善操作）
- 中枢之上（未出现三买）→ 等待三买确认
- 中枢之上（已出现三买）→ 持有或趋势判断
- 中枢之下（未出现三卖）→ 观望
- 中枢之下（已出现三卖）→ 在卖点附近减仓

#### 第六层：多级别联立

`观察者` 内部维护了完整的**多级别管道**，在每一根新 K 线到达时**一次性计算**所有级别的全部结构：

| 序列 | 含义          | 基础元素 |
|------|-------------|----|
| 笔序列 | 笔           | 分型 |
| 线段序列 | 线段          | 笔 |
| 扩展线段序列 | 扩展线段        | 笔 |
| 线段-线段序列 | 以线段为元素的线段   | 线段 |
| 扩展线段-线段序列 | 以线段为元素的扩展线段 | 线段 |
| 扩展线段-扩展线段序列 | 以扩展线段为元素的扩展线段 | 扩展线段 |
| 笔中枢序列 | 笔中枢         | 笔 |
| 线段中枢序列 | 线段中枢        | 线段 |
| ... | 其余各级别中枢     | 对应级别 |

**扩展线段**，趋近于《教你炒股票 54：一个具体走势的分析》同级别分解的例子。

这种设计使得**跨级别分析**成为可能——你可以同时看到一个走势在笔级别、线段级别、以及更高级别的结构演化。

## 四、技术指标体系

所有指标支持**增量计算**（基于前一根递推），避免重复扫描。

### 4.1 MACD（平滑异同移动平均线）

- **快线 EMA / 慢线 EMA**：EMA = α × 当前价 + (1-α) × 前值，α = 2/(N+1)
- **DIF** = 快线 EMA - 慢线 EMA
- **DEA** = EMA(DIF, 信号周期)
- **MACD 柱** = DIF - DEA

### 4.2 RSI（相对强弱指数）

- 使用 **Wilder 平滑**（RMA）而非 SMA
- 上涨幅度 = max(Δ收盘价, 0)，下跌幅度 = max(-Δ收盘价, 0)
- RS = 平均上涨 / 平均下跌，RSI = 100 - 100/(1+RS)
- 可选 RSI_SMA：对 RSI 值计算简单移动平均作为信号线

### 4.3 KDJ（随机指标）

- **RSV** = (收盘价 - N日最低) / (N日最高 - N日最低) × 100
- **K** = 2/3 × 前K + 1/3 × 当日 RSV（M1=3 时）
- **D** = 2/3 × 前D + 1/3 × 当日 K（M2=3 时）
- **J** = 3K - 2D

## 五、背驰（背离）分析

`背驰分析` 类（第 1160 行）提供四种背驰判断方法：

### 5.1 MACD 背驰

比较进入段和离开段的 MACD 柱面积（阳柱之和 / 阴柱之和 / 总绝对值）。面积衰减即为背驰。

### 5.2 斜率背驰

比较两段的价格变化率（Δ价格 / Δ时间）。价格创新高/新低但斜率绝对值减小即为背驰。

### 5.3 测度背驰

比较两段的**欧几里得距离** sqrt(Δx² + Δy²)，将时间和价格统一到"测度"维度。测度缩短但价格极值超越即为背驰。

### 5.4 组合模式

| 模式 | 逻辑 |
|------|------|
| 全量背驰 | MACD ∧ 测度 ∧ 斜率全部满足 |
| 任意背驰 | MACD ∨ 测度 ∨ 斜率任一满足 |
| 任选背驰 | 三种中至少两种满足（多数投票） |
| 配置背驰 | 按缠论配置中的开关组合判断 |

## 六、配置系统

`缠论配置`（第 253 行）基于 Pydantic `BaseModel`，提供约 80 个配置字段，涵盖：

- **分析开关**：`分析笔`、`分析线段`、`分析扩展线段`、`分析笔中枢`、`分析线段中枢`
- **笔参数**：`笔内元素数量`、`笔弱化`、`笔次级成笔`、`笔内相同终点取舍`
- **线段参数**：`线段_特征序列忽视老阴老阳`、`线段_缺口后紧急修正`、`线段_非缺口下穿刺`、`线段_修正`
- **指标参数**：MACD 三个周期、RSI 周期与阈值、KDJ 参数
- **买卖点参数**：`买卖点偏移`、`买卖点_背离率`、各种买卖点识别开关
- **展示参数**：各种图显开关

配置支持 JSON 序列化 / 反序列化，自带向前兼容（多余字段忽略，缺少字段使用默认值）。

## 七、高阶组件

### 7.1 K线合成器（`K线合成器`, 第 3402 行）

将低周期 K 线（如 1 分钟）合成为高周期 K 线（如 5 分钟、30 分钟）。支持：
- 多周期并行合成
- 时间戳对齐到周期边界
- 完成通知回调（`事件回调`）
- 增量更新（开、高、低、收、量）

### 7.2 立体分析器（`立体分析器`, 第 3505 行）

`K线合成器` + 多个 `观察者` 的组合体。接收最小周期的原始 K 线，自动合成高周期 K 线，并反馈给对应周期的观察者。实现了**真正的多级别联立**——在同一个数据流下，各周期独立分析但共享缠论 K 线序列。

### 7.3 买卖点系统（第 122-206 行）

支持**传统三类买卖点**（一买 / 一卖 / 二买 / 二卖 / 三买 / 三卖）和**缠论六类买卖点**（T1/T1P/T2/T2S/T3A/T3B 及其买卖方向），共 18 种买卖点类型。

买卖点对象（`基础买卖点`）记录：
- 买卖点分型与 K 线
- 偏移量（从买卖点到当前 K 线的距离）
- 失效偏移
- 中枢破位值
- 与 MACD / RSI / KDJ 的匹配状态

## 八、关键设计细节

### 8.1 递归深度守卫

`笔.分析` 和 `线段.分析` 均设置了递归深度上限（64 和 256）。这是由于"变动即重建"策略在极端数据下可能触发连锁反应式回退。深度守卫防止无限递归。

### 8.2 事后修复机制

`笔.分析`（第 2039-2047 行）在弹出旧笔后，会尝试从弹出位置扫描后续 K 线，补回可能被"错过"的笔——称为"事后修复"。这处理了因分型序列重排导致的中间笔丢失问题。

### 8.3 分割序列的第三买卖点识别

`线段.分割序列`（第 2355 行）除了分割前/后序列外，还会识别线段终点相对于中枢的位置：
- **中枢之上** + 向下的笔组与中枢存在向上缺口 → 标记三买
- **中枢之下** + 向上的笔组与中枢存在向下缺口 → 标记三卖

### 8.4 中枢扩展

`中枢.获取扩展中枢`（第 3042 行）在中枢基础序列 ≥ 9 时，对其包含的笔/线段做扩展线段分析，进而产生"扩展中枢"。这是缠论中盘整走势 → 趋势转化的关键机制。
